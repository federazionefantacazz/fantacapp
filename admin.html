SofaScore Sync

⚡ Sincronizzazione automatica voti

Questo pannello mostra come configurare il fetch automatico dei voti da SofaScore.
SofaScore non ha un'API pubblica ufficiale ma espone endpoint non documentati usati dalla loro stessa app mobile. Lo script qui sotto li usa per prendere le valutazioni dei giocatori dopo ogni partita.

In attesa…
📡 Come funziona il sync automatico

Per avere i voti completamente automatici senza che tu faccia niente, usa uno di questi approcci:

🥇 Opzione 1 — GitHub Actions (GRATIS, consigliata)
Crea un repository GitHub con il file .github/workflows/sync.yml allegato qui sotto. GitHub esegue automaticamente lo script Python ogni 30 minuti nei giorni di partita. Lo script fetcha SofaScore e scrive i voti su Firebase. Costo: 0€.
🥈 Opzione 2 — Raspberry Pi / PC sempre acceso
Esegui lo script Python con un cronjob ogni 30 minuti. Funziona su qualsiasi computer Linux/Mac/Windows.
🥉 Opzione 3 — Sync manuale da qui
Clicca "Avvia sync manuale" dopo ogni giornata. I voti vengono scaricati da SofaScore e salvati su Firebase. Tutti i telefoni si aggiornano in meno di 1 secondo.
📋 Script Python (sync.py)
Salva questo file e impostalo come cronjob o GitHub Action:
import requests, firebase_admin from firebase_admin import credentials, db import json, os # --- CONFIG --- FIREBASE_URL = "https://TUOPROGETTO-default-rtdb.europe-west1.firebasedatabase.app" SERIE_A_ID = 23 # ID Serie A su SofaScore cred = credentials.Certificate("serviceAccountKey.json") firebase_admin.initialize_app(cred, {"databaseURL": FIREBASE_URL}) def get_serie_a_matches(): url = f"https://api.sofascore.com/api/v1/sport/football/scheduled-events/today" headers = {"User-Agent": "Mozilla/5.0"} r = requests.get(url, headers=headers, timeout=10) events = r.json().get("events", []) return [e for e in events if e.get("tournament",{}).get("id") == SERIE_A_ID] def get_player_ratings(match_id): url = f"https://api.sofascore.com/api/v1/event/{match_id}/lineups" headers = {"User-Agent": "Mozilla/5.0"} r = requests.get(url, headers=headers, timeout=10) data = r.json() ratings = {} for side in ["home", "away"]: for p in data.get(side, {}).get("players", []): pid = p.get("player", {}).get("id") rating = p.get("statistics", {}).get("rating") if pid and rating: ratings[str(pid)] = round(float(rating), 1) return ratings def sync(): matches = get_serie_a_matches() print(f"Trovate {len(matches)} partite Serie A") all_ratings = {} for m in matches: mid = m["id"] status = m.get("status", {}).get("type", "") # Aggiorna partite live su Firebase db.reference(f"liveMatches/{mid}").set({ "homeTeam": m.get("homeTeam",{}).get("shortName","?"), "awayTeam": m.get("awayTeam",{}).get("shortName","?"), "homeScore": m.get("homeScore",{}).get("current"), "awayScore": m.get("awayScore",{}).get("current"), "minute": m.get("time",{}).get("played"), "status": status, }) # Prendere i voti solo per partite finite o in corso if status in ("inprogress", "finished"): ratings = get_player_ratings(mid) all_ratings.update(ratings) print(f" Match {mid}: {len(ratings)} voti") # Scrivi i voti su Firebase (ogni player: sofaId -> voto) if all_ratings: db.reference("votes").update(all_ratings) print(f"Salvati {len(all_ratings)} voti su Firebase") if __name__ == "__main__": sync()
Dipendenze: pip install requests firebase-admin
⚙️ GitHub Actions (sync.yml)
name: SofaScore Sync on: schedule: - cron: '*/30 * * * 5,6,0' # ogni 30min ven/sab/dom workflow_dispatch: # eseguibile manualmente jobs: sync: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-python@v5 with: {python-version: '3.11'} - run: pip install requests firebase-admin - run: python sync.py env: GOOGLE_APPLICATION_CREDENTIALS: serviceAccountKey.json
